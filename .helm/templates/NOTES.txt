paibot is deployed in namespace {{ .Release.Namespace }}.

It connects to Slack via Socket Mode (outbound WebSocket) — no ingress or service needed.

Check status:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "paibot.name" . }}

View logs:
  kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "paibot.name" . }} -f
{{- if not .Values.existingSecret }}

WARNING: existingSecret is not set. The bot will fail to start without
SLACK_APP_TOKEN, SLACK_BOT_TOKEN, OPENAI_BASE_URL, OPENAI_API_KEY, and OPENAI_MODEL.

Create the secret first:
  kubectl create secret generic <secret-name> -n {{ .Release.Namespace }} \
    --from-literal=SLACK_APP_TOKEN=xapp-... \
    --from-literal=SLACK_BOT_TOKEN=xoxb-... \
    --from-literal=OPENAI_BASE_URL=https://... \
    --from-literal=OPENAI_API_KEY=... \
    --from-literal=OPENAI_MODEL=...

Then set existingSecret=<secret-name> in your values.
{{- end }}
